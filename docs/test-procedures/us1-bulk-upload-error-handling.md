## 边界情况测试 (Edge Cases)

### 测试场景：空文件夹处理
1. 创建一个空文件夹（不包含任何文件）
2. 在批量上传中选择该空文件夹
3. 验证系统显示"未在所选目录中找到 JSON 文件"提示
4. 确认界面保持可用状态

### 测试场景：仅非JSON文件文件夹
1. 创建包含 .txt、.md、.png 等非JSON文件的文件夹
2. 执行批量上传选择
3. 验证系统过滤掉所有非JSON文件
4. 检查提示信息准确显示忽略的文件类型和数量

### 测试场景：超大JSON文件处理
1. 准备一个超过10MB的JSON文件
2. 尝试上传该文件
3. 验证系统拒绝处理并显示"文件过大"错误信息
4. 确认其他有效文件仍能正常处理

### 测试场景：网络连接中断
1. 开始批量上传作业
2. 在上传过程中断开网络连接
3. 重新连接网络后检查作业状态
4. 验证系统能够恢复或正确报告中断状态

### 测试场景：权限不足情况
1. 在权限受限的环境中尝试文件夹选择
2. 验证系统显示适当的错误提示
3. 检查降级方案（文件选择）是否可用

### 测试场景：混合文件类型批量
1. 准备包含 50个有效JSON、20个无效JSON、10个非JSON文件的混合文件夹
2. 执行批量上传
3. 验证统计信息准确：
   - 有效文件数量：50
   - 无效文件数量：20
   - 忽略文件数量：10
4. 检查错误详情包含所有失败文件的具体信息

### 测试场景：重复文件名处理
1. 在同一批次中包含多个同名文件
2. 验证系统能够正确处理文件名冲突
3. 检查每个文件都被单独处理

### 测试场景：特殊字符文件名
1. 使用包含中文、空格、特殊符号的文件名
2. 验证文件名正确显示和处理
3. 确认数据库存储和检索正常

### 测试场景：嵌套文件夹深度测试
1. 创建深度超过5层的嵌套文件夹结构
2. 验证递归扫描能够处理所有层级
3. 检查相对路径信息正确记录
