# 测试流程：排行榜显示格式 (US1)

**用户故事**: US1 - 查看使用排行榜
**测试目标**: 验证排行榜的视觉显示格式正确，包括排名、奖牌、数值格式化等
**测试环境**: 小程序开发工具 / 真机

## 前置条件

1. 已完成使用排行榜功能测试
2. 小程序排行榜页面可以正常加载数据
3. 至少有5个剧本数据用于测试显示格式

## 测试步骤

### 排名显示测试

1. **数字排名显示**
   - 检查第4名及以后显示数字排名（4, 5, 6...）
   - 验证数字字体大小和颜色正确
   - 检查数字居中对齐

2. **奖牌显示测试**
   - 验证前三名显示奖牌图标：
     - 第1名：🥇 金色奖牌
     - 第2名：🥈 银色奖牌
     - 第3名：🥉 铜色奖牌
   - 检查奖牌图标大小合适
   - 验证奖牌颜色与背景协调

3. **排名样式一致性**
   - 检查所有排名徽章大小一致（80rpx × 80rpx）
   - 验证排名徽章圆形显示
   - 检查数字和奖牌都在徽章中央

### 剧本信息显示测试

4. **标题显示**
   - 检查剧本标题完整显示
   - 验证标题字体大小和颜色
   - 测试长标题的截断处理（如超过2行显示省略号）

5. **作者显示**
   - 验证"作者："前缀正确显示
   - 检查作者姓名显示完整
   - 验证作者信息颜色和字体

6. **数值显示格式**
   - 检查使用次数数字正确显示
   - 验证"次使用"标签显示
   - 测试大数字的显示（千位分隔符，如1,250）

### 布局和交互测试

7. **列表项布局**
   - 验证每个列表项高度一致
   - 检查排名徽章、剧本信息、数值区域的相对位置
   - 测试不同屏幕尺寸下的布局适配

8. **点击区域测试**
   - 验证整个列表项都是可点击区域
   - 检查点击反馈效果（背景色变化）
   - 确认点击区域不包含排名徽章（避免误操作）

9. **视觉层次测试**
   - 检查重要信息（标题、排名）的视觉权重
   - 验证次要信息（作者、数值）的视觉层次
   - 测试颜色对比度和可读性

### 响应式测试

10. **不同设备适配**
    - 在不同屏幕尺寸设备上测试
    - 验证文字大小在小屏幕上的可读性
    - 检查列表项在窄屏幕上的布局

11. **深色模式适配**（如果支持）
    - 测试深色模式下的颜色显示
    - 验证对比度仍然符合可读性标准
    - 检查图标和文字在深色背景下的显示效果

## 验收标准

### ✅ 通过标准

**排名显示**
- [ ] 前三名正确显示奖牌图标🥇🥈🥉
- [ ] 其他名次显示数字排名
- [ ] 排名徽章样式一致，大小合适

**信息显示**
- [ ] 剧本标题清晰可读
- [ ] 作者信息格式正确
- [ ] 数值和单位标签显示完整

**布局和交互**
- [ ] 列表项布局整齐美观
- [ ] 点击区域明确，反馈明显
- [ ] 视觉层次清晰，信息易读

**响应式适配**
- [ ] 在不同设备上显示正常
- [ ] 文字在小屏幕上仍然可读
- [ ] 布局在各种屏幕尺寸下都合理

### ❌ 失败标准

- [ ] 排名显示混乱或错误
- [ ] 文字显示不完整或难以阅读
- [ ] 布局错乱或元素重叠
- [ ] 点击区域不明确或无反馈
- [ ] 在某些设备上显示异常

## 特殊测试用例

### 长文本处理
- 剧本标题超过20个字符
- 作者姓名特别长的情况
- 数值特别大的情况（百万级）

### 边界排名
- 只有1个剧本的情况
- 只有2-3个剧本的情况
- 超过20个剧本的情况

### 特殊字符
- 剧本标题包含特殊字符
- 作者姓名包含emoji
- 数值格式的各种情况

## 性能考虑

- 列表渲染应该流畅
- 大量数据时滚动性能良好
- 内存使用合理

## 回归检查清单

修改样式或布局后需要检查：
- [ ] 排名徽章位置是否正确
- [ ] 文字是否仍然可读
- [ ] 点击区域是否仍然有效
- [ ] 不同设备上的显示是否正常
- [ ] 动画效果是否流畅

## 设计规范对照

根据Ant Design设计规范：
- 文字颜色：主要文字 #333，次要文字 #666
- 间距：元素间距16rpx，内部间距24rpx
- 圆角：8rpx
- 阴影：0 4rpx 16rpx rgba(0,0,0,0.08)

## 备注

- 所有视觉测试都需要在真机上验证
- 建议使用设计工具对比设计稿和实际效果
- 记录任何与设计稿的偏差并说明原因
