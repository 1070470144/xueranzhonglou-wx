# 优化投稿须知界面文字格式 - 需求清单

## 功能需求

### 核心需求
- [x] **文字格式化**: 为投稿须知界面的文字内容提供适当的格式层次
- [x] **组件化结构**: 将纯文本拆分为标题、段落、列表等专用组件
- [x] **视觉层次**: 不同类型内容有明显的视觉区别
- [x] **交互优化**: 邮箱地址支持点击复制功能

### 技术需求
- [x] **uni-app兼容**: 使用uni-app组件和CSS样式，不使用HTML标签
- [x] **响应式设计**: 在不同屏幕尺寸下保持良好的显示效果
- [x] **动画保持**: 保持原有的页面进入动画效果
- [x] **性能优化**: 不影响页面加载速度和渲染性能

## 非功能需求

### 用户体验需求
- [x] **可读性提升**: 内容层次清晰，易于阅读和理解
- [x] **扫描友好**: 支持用户快速浏览重要信息
- [x] **移动端适配**: 在小屏幕设备上保持良好的阅读体验
- [x] **操作反馈**: 点击复制邮箱时有明确的用户反馈

### 视觉设计需求
- [x] **字体层级**: 标题(30rpx)、正文(28rpx)的明确大小区别
- [x] **颜色规范**: 标题(#333)、正文(#555)、强调(#007AFF)的合理配色
- [x] **间距规范**: 段落间距(32rpx)、列表项间距(16rpx)的统一规范
- [x] **列表样式**: 有序列表数字编号、无序列表圆点符号
