# 新增：我的页面（无登录）与“投稿要求”信息面板

## 概述

在应用中新增一个“我的”页面（无需登录），页面包含一个功能项：`投稿要求`。点击该项打开一个信息面板（侧栏或模态对话框），展示投稿说明内容（内容暂定，可由产品填写）。该功能仅为展示信息，不要求用户提交或认证。

## 关键概念
- Actor: 匿名移动端用户、内容投稿者（潜在）、产品人员
- Action: 访问“我的”页面，点击“投稿要求”，查看信息面板内容，关闭面板
- Data: 投稿说明文本（只读）
- Constraint: 无需登录或权限校验；需兼容微信小程序与 HBuilderX 预览

## 用户场景

1. 用户打开应用底部或侧边导航，进入“我的”页面（无需登录）。
2. 页面展示若干功能入口，其中包含“投稿要求”一项。
3. 用户点击“投稿要求”，页面弹出信息面板（模态或底部上拉/侧滑），显示投稿说明文本，用户可阅读并关闭面板。
4. 面板关闭后返回“我的”页面，状态无变化。

## 功能需求（Functional Requirements）

FR1 — 我的页面（无登录）
- 页面路径为 `pages/my/my.vue`（或 app 指定位置），用户无需登录可访问。
- 页面应包含至少一个交互入口：`投稿要求`。

FR2 — 打开信息面板
- 点击 `投稿要求` 后打开信息面板（模态/侧栏），展示可滚动的文本内容，支持长文本的阅读与回退关闭操作（右上角关闭或背景点击）。
- 面板应包含标题（例如“投稿要求”）和正文区域，正文内容占位“内容暂定”。

FR3 — 可访问性与响应式
- 面板在不同屏幕尺寸下应可读、可滚动，文字大小与行距满足移动阅读体验。
- 面板在无网络或离线场景仍可打开（内容为本地静态文本）。

FR4 — 无数据提交与无登录
- 功能仅用于展示信息，不涉及表单提交或后台交互，不应触发登录流程。

FR5 — 可本地更新内容
- 投稿说明文本应放置在本地静态资源或可由配置文件更新，而非硬编码在组件内部（便于产品方日后修改）。

## 验收标准（Acceptance Criteria）

AC1 — 页面可访问
- 未登录用户打开应用后能访问 `pages/my/my` 页面且不被重定向到登录。

AC2 — 信息面板展示
- 点击“投稿要求”后，面板打开并展示标题与正文（正文含“内容暂定”占位文本），用户可以滚动阅读并通过关闭按钮或点击背景关闭面板。

AC3 — 离线可用
- 在断网情况下打开页面及面板，正文仍能显示（来自本地静态资源）。

AC4 — 无副作用
- 打开或关闭面板不应触发网络请求或登录流程；页面状态（如滚动位置）按标准行为返回。

## 关键实体
- MyPage: 页面视图（包含功能入口列表）
- InfoPanel: 带标题与正文的模态/侧栏组件
- StaticContent: 本地静态文本资源（投稿说明）

## 假设与约束

- 假设前端可新增页面与组件文件；不改动后端。
- 假设投稿说明内容为只读文本，产品方提供最终文本后替换占位内容。
- 约束：兼容微信小程序与 HBuilderX，避免使用受限平台 API。

## 验证测试计划

- 单元测试：组件渲染测试（MyPage、InfoPanel），点击触发面板打开/关闭，断网场景下展示本地内容。
- UI/视觉测试：在常见设备宽度下截图，验证面板显示与滚动行为。
- 手工验收：产品确认投稿说明内容并验收视觉与交互。

## 可交付项

1. 页面组件：`pages/my/my.vue`（带“投稿要求”入口）
2. 信息面板组件：`components/InfoPanel.vue`（可复用）
3. 本地静态内容文件：`static/content/submission.md` 或 `src/config/submission.js`
4. 单元测试：`tests/pages/my/my.test.js` 与组件测试
5. 视觉截图与验收记录
