# 研究记录：替换为 uni-app 自带回退按钮

**日期**：2026-01-12  
**Feature**：`specs/7-uni-back-btn/spec.md`

## 研究任务

1. 验证 uni-app 是否提供合适的回退图标类型（如 `left`, `arrow-left`, `chevron-left`）以及 `uni.navigateBack()` 在目标平台上的一致性。  
2. 评估在浅色头部背景下的可见性策略（细边框 / 浅填充 / 加深图标色）。  
3. 确认不引入第三方图标库，保持内置组件优先。

## 结论与决策

- 决策：使用内置 `uni-icons`（优先类型 `chevron-left` 或 `left`），并用 `uni.navigateBack()` 实现导航回退。  
- 在浅色头部采用轻微半透明填充或浅边框以提高可见性（实现方案可通过 CSS 变量切换）；在图片头图或深色头图下保留默认视觉或轻阴影提升对比。  
- 不引入外部图标库或额外依赖，所有调整以 CSS 实现。

## 替代方案

- 使用外部 SVG/图标库（已拒绝，违背 UNI-APP-FIRST）  
- 使用平台原生图标（放弃以保持 uni-app 跨平台一致性）

## 实施注记

- 实现时需同时在 HBuilderX 与微信模拟器验证外观差异，并把视觉基线覆盖浅/深背景两类场景。
